Feature: NFR-01 Производительность API создания тренировок
  Scenario: время ответа на создание тренировки находится в пределах нормы
    Given API-сервис запущен и доступен для нагрузочного тестирования
    When  клиент выполняет 5-минутный тест с нагрузкой 50 RPS на эндпоинт POST /workouts/
    Then  p95 времени ответа не превышает 250 мс

Feature: NFR-03 Валидация вводимых данных для подходов (сетов)
  Scenario: система успешно принимает корректные данные о подходе
    Given аутентифицированный пользователь существует
    And   в системе есть тренировка с ID "workout-123" и упражнение с ID "exercise-456"
    When  пользователь отправляет POST-запрос на /workouts/workout-123/sets с телом `{"reps": 10, "weight": 50.5}`
    Then  система отвечает статусом 200 OK и возвращает обновленные данные тренировки

Feature: NFR-03 Валидация вводимых данных для подходов
  Scenario: система отклоняет некорректные (отрицательные) данные о подходе
    Given аутентифицированный пользователь существует
    And   в системе есть тренировка с ID "workout-123"
    When  пользователь отправляет POST-запрос на /workouts/workout-123/sets с телом `{"reps": -5, "weight": 10}`
    Then  система отвечает статусом 422 Unprocessable Entity
